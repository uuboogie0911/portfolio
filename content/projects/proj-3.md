# 추가 정보

## 주요 기능

### 운영 대시보드
- 항공사별 데이터 소스 상태 모니터링
- 실시간 장애 감지 및 알림
- 데이터 정합성 검증 도구

### Fallback 시스템
- 특정 소스 장애 시 자동 대체 소스 전환
- 전체 장애 시 예약 차단 및 안내 정책
- 서비스 연속성 보장 메커니즘

### 데이터 소스 관리
- 항공사별 우선순위 정책 설정
- 데이터 충돌 시 적용 규칙(Rules) 관리
- 소스별 특성 분석 및 최적화

## 데이터 분석 결과

### 플랫폼 안정성
- 데이터 정합성 문제 감소
- 소스 장애 시에도 서비스 연속성 유지
- 운영 및 CS 이슈 감소

### 확장성 확보
- 신규 항공사·공급사 연동 기반 마련
- 표준화된 연동 프로세스 구축

### 운영 효율
- 운영 효율 고도화
- 자동화 기반 확보

## 주요 의사결정

### 플랫폼 아키텍처 설계
- 단순 연동이 아닌 플랫폼 레벨에서 데이터 판단 및 선택 구조 설계
- "플랫폼 레벨에서 데이터를 어떻게 판단하고 선택할 것인가"를 중심으로 설계

### 데이터 소스 정책 수립
- 항공 데이터 소스별 특성을 분석하고 우선순위 정책 정의
- 데이터 충돌 시 적용할 기준 규칙(Rules) 수립

### 장애 대응 체계
- 특정 소스 장애 발생 시 대체 소스를 자동으로 사용하는 fallback 구조 설계
- 전체 장애 시 잘못된 예약이 발생하지 않도록 예약 차단 및 안내 정책 마련

## 프로젝트 성과

### 비즈니스 임팩트
- 항공사/OTA 표준화 및 자동화 기반 확보
- 운영 효율 고도화
- 향후 확장 가능한 플랫폼 구조 확보

### 기술적 성과
- 국내 최초 NDC Aggregator 플랫폼 운영 대시보드 구축
- 데이터 정합성 문제 해결
- 서비스 안정성 및 확장성 확보

### 운영 개선
- 운영 및 CS 이슈 감소
- 표준화된 운영 프로세스 정립
- UAT 검증을 통한 품질 보증
