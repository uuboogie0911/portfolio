# B2B 원단 플랫폼 키위블랙

## 서비스 요약
패션 브랜드/공방/소호몰 등 원단 구매자를 대상으로, 온라인에서 원단 탐색 → 스와치 요청 → 주문 → 출고/정산 관리까지 가능한 B2B 플랫폼 신규 구축.

## 개발기간
서비스 오픈 3개월 + 고도화 6개월

## 투입인원
기획 1명, 디자인 1명, 개발 1명

## 🚩 문제정의
- 원단 정보 오프라인 중심 → 재고/가격 확인 비효율
- 견적+주문+출고 프로세스 단절 / 전화·카톡 수동 처리
- 정형화되지 않은 품목/단위/옵션 체계 → 신규 사용자 이해 어려움
- 전산화가 안 된 OMS/WMS → 운영 리소스 과소비

## 🎯 목표
- 원단 재고/옵션/스와치 정보 온라인 전환
- OMS/WMS 연동으로 주문–출고 자동화
- 구매자가 이해 가능한 구조로 콘텐츠/분류 개편
- CS 의존 업무 → 셀프 처리율 증가

## 🔄 FLOW 구조

### Before
전화/메신저 문의 → 수동 재고 확인 → 종이/엑셀 견적 → 별도 배송 요청

### After
상품 탐색 → 스와치 요청 → 단가/재고 자동 확인 → 주문/결제 → 출고 연동

## 🧩 핵심 의사결정
- 서비스 구조 단순화: 스와치/재단/롤단 단위 표준화
- 카테고리/텍스타일 정보 → 전시 기준/필터링 재정의
- 주문~출고 연결되는 OMS/WMS 구축
- 매입/매출/정산 프로세스 분리 운영

## 📈 성과
- 스와치 요청/출고 처리 속도 단축
- 신규 문의 대비 셀프 정보 탐색 비중 증가
- 운영 핸드오버 가능 수준의 업무 표준화 완료

## 🧑‍💻 역할
- 신규 사업 PM / 초기 제로베이스 IA/UX 설계
- 정책/요구사항/스토리보드/QA
- 콘텐츠/분류/카테고리/상품 구조 재정리
- OMS/WMS 처리 정책 + 예외 시나리오 정립
