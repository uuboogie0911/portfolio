# 추가 정보

## 주요 화면

### 로그인 화면
- 간편 로그인 옵션 제공 (이메일, 휴대폰, SNS)
- 비회원 구매 옵션 노출
- 로그인 상태에 따른 메인 전시 차별화

### 회원가입 화면
- 단계별 가입 프로세스 (간편회원 → 정회원)
- 최소 정보 입력으로 간편 가입 가능
- 결제 시점에만 간편회원 생성

### 메인 화면
- 로그인 상태에 따라 CTA 및 안내 메시지 다르게 노출
- 비로그인 사용자도 핵심 기능 이용 가능

## 데이터 분석 결과

### 전환율 개선
- 신규 가입률: **100% 증가**
- 회원가입 전환율: **약 2배 이상 개선**
- 로그인 이탈률: **24% 감소**

### 사용자 행동 변화
- 비로그인 사용자 재방문 증가
- 서비스 가치 경험 후 가입으로 이어지는 패턴 확인
- CS 문의 감소 및 운영 효율 개선

## 주요 의사결정

### 회원 상태 정의
- **비로그인**: 탐색 및 핵심 기능 이용 가능
- **간편회원**: 최소 정보 기반, 결제 시점 생성
- **정회원**: 결제 완료 후 전환 유도

### 인증 시점 설계
- "언제 인증이 필요한가"를 기준으로 정책과 UX 함께 설계
- 탐색 단계: 로그인 불필요
- 결제 단계: 간편회원 생성
- 결제 후: 정회원 전환 유도

## 프로젝트 성과

### 비즈니스 임팩트
- 회원가입 전환율 대폭 개선으로 신규 유저 확보 증가
- 로그인 이탈률 감소로 사용자 경험 개선

### 운영 효율성
- 회원 상태 명확화로 CS 응대 효율 개선
- 내부 운영 혼선 감소
- 정책 문서화로 일관성 확보
